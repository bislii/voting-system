Функциональные требования
Создание опросов.
Система должна позволять зарегистрированным пользователям создавать новые опросы.
При создании опроса пользователь должен указать заголовок (вопрос) опроса.
Пользователь должен иметь возможность добавить от 2 до 10 вариантов ответа для каждого опроса.
Пользователь может установить дату и время завершения голосования.
1
Просмотр опросов.
Система должна отображать список всех активных опросов.
Система должна отображать подробную информацию о каждом опросе (заголовок, варианты ответа, статус - активен/завершен, дата завершения).

Голосование.
Система должна позволять зарегистрированным пользователям голосовать в активных опросах.
Система должна гарантировать, что каждый зарегистрированный пользователь может проголосовать в конкретном опросе только один раз.
После голосования пользователь должен видеть результаты опроса.

Отображение результатов.
Система должна отображать текущие результаты голосования (например, в процентах или количестве голосов) после того, как пользователь проголосовал.
После завершения голосования система должна отображать окончательные результаты для всех пользователей.

Управление пользователями.
Система должна предоставлять функции регистрации новых пользователей (например, по email и паролю).
Система должна предоставлять функции аутентификации (входа) существующих пользователей.
Пользователь должен иметь возможность просматривать список опросов, которые он создал или в которых голосовал (опционально для простоты).

Нефункциональные требования.

Производительность.
Время загрузки страницы со списком опросов не должно превышать 3 секунд при стандартной нагрузке.
Время обработки одного голоса не должно превышать 1 секунды.

Безопасность.
Система должна защищать от попыток накрутки голосов или многократного голосования одного пользователя.
Данные пользователей (например, пароли) должны храниться в зашифрованном виде.
Система должна быть устойчива к базовым видам веб-атак (XSS, SQL-инъекции и т.д.).

Удобство использования.
Пользовательский интерфейс должен быть интуитивно понятным для выполнения основных действий (регистрация, создание опроса, голосование).
Система должна предоставлять четкую обратную связь пользователю (например, подтверждение голоса, уведомление об ошибке).

Надежность.
Система должна быть доступна для пользователей 99% времени (исключая плановое обслуживание).
Должны быть реализованы механизмы обработки ошибок при сбоях (например, при потере интернет-соединения).

Совместимость.
Веб-интерфейс должен корректно отображаться и функционировать в основных браузерах Chrome, Firefox (Всё), и на настольных и мобильных устройствах.
Масштабируемость (для простого приложения - минимальные требования):
Архитектура системы должна допускать возможность обработки умеренного увеличения числа пользователей и опросов в будущем без полной переработки.

Поддерживаемость.
Код системы должен быть структурирован, иметь комментарии и документацию для облегчения дальнейшей поддержки и развития.
